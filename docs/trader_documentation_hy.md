# Կրիպտո-ֆյուչերսների ավտոմատ սիգնալային համակարգի բացատրություն

Այս փաստաթուղթը գրված է թրեյդերի համար։ Նպատակը՝ պարզ լեզվով բացատրել, թե ինչպես է ծրագիրը գտնում ուղղությունը (Long/Short), ինչպես է որոշում stop-loss և take profit մակարդակները, և ինչ մաթեմատիկայի ու մոդելների վրա է դա հիմնված։

## 1. Ի՞նչ է անում համակարգը ընդհանուր առմամբ

Համակարգը աշխատում է Binance-ի USDT ֆյուչերսների գների վրա և ամեն րոպե՝
- հետ է բերում վերջին ~50 մոմերը (1-րոպեանոց գրաֆիկ),
- հաշվարկում է մի շարք տեխնիկական ցուցիչներ,
- որոշում է՝ կա՞ արդյոք հետաքրքիր հնարավորություն Long կամ Short դիրքի համար,
- եթե կա, առաջարկում է մուտքի գին (entry), stop-loss և 3 նպատակային մակարդակ (TP1, TP2, TP3),
- հետո հետևում է գնի շարժին և գրանցում է՝ արդյոք այդ սիգնալը հաջողվեց (TP) թե ձախողվեց (SL).

Ծրագիրը կարող է տեղադրել օրդերներ ավտոմատ, բայց սիգնալների հաշվարկն ու ստուգումը աշխատում են նույնիսկ առանց API բանալիների։ Այսինքն՝ միայն որպես «սիգնալ գեներատոր» նույնպես լիովին օգտակար է։

## 2. Ինչ տվյալներ ու ցուցիչներ են օգտագործվում

Համակարգը հաշվում է հետևյալ հիմնական ցուցիչները՝ վերջին րոպեների գների հիման վրա.

- RSI (Relative Strength Index)
  Ցույց է տալիս, թե վերջին շարժը ավելի շատ գնային բարձրացո՞ւմ է եղել, թե՞ իջեցում։ Բարձր RSI՝ գերգնված շուկա, ցածր RSI՝ գերբացառված։

- Մոմենտում և աքսելերացիա
  Մոմենտումը չափում է, թե որքան արագ է փոխվում գինը (արագություն), աքսելերացիան՝ փոփոխության արագության արագացումը։ Դրանք օգնում են տեսնել՝ շարժը թուլանու՞մ է, թե՞ ուժեղանում։

- Վոլատիլություն և ATR (Average True Range)
  Վոլատիլությունն ու ATR-ը չափում են, թե միջինում որքան «լայն» են մոմերը՝ ինչքան են գները տատանվում վերջին ժամանակահատվածում։ Սա օգտագործվում է stop-loss-ի և թարգեթների հեռավորությունը որոշելու համար, որպեսզի դրանք ոչ շատ մոտ, ոչ էլ չափազանց հեռու լինեն։

- Շուկայի կառուցվածք (structure) և range/trend դասակարգում
  Պարզ ալգորիթմով ծրագիրը մոտավոր հասկանում է՝ շուկան ավելի շատ թրենդային է (շարժվում է հիմնականում վերև կամ ներքև), թե՞ range-ի մեջ է (շարժվում է լայն միջակայքում առաջ-ետ): Սա նույնպես ներմուծվում է մոդել, որպես լրացուցիչ ինֆորմացիա։

- Fibonacci մակարդակներ
  Հաշվում են նաև որոշ հիմնական Fibonacci մակարդակներ (օրինակ՝ 0.382, 0.5, 0.618): Դրանք հիմա հիմնականում օգտագործվում են որպես վերլուծական ինֆորմացիա լոգերում, ոչ թե ուղղակիորեն մոդելի որոշման մեջ։

## 3. Ինչպես է որոշվում սիգնալի ուղղությունը (Long / Short)

Ուղղության որոշումը արվում է երկու քայլով՝

1) Պարզ կանոնային մոդել (rule-based)
- Օգտագործվում է շարժվող միջինների (Moving Average) հեշտ կանոն՝
  - հաշվում է վերջին 5 մոմի ու վերջին 20 մոմի միջին գները,
  - եթե կարճ միջինը (5) բարձր է երկար միջինից (20) → կարծվում է, որ շուկան ավելի շատ բարձրացող է → BUY (Long) սիգնալ,
  - եթե կարճ միջինը ցածր է երկար միջինից → SELL (Short) սիգնալ,
  - եթե տարբերությունը չնչին է → NO_TRADE (ոչ մի սիգնալ):

2) Մեքենայական ուսուցման մոդել (ML)
- Օգտագործվում է մի խումբ decision-tree տիպի մոդելներ, որոնք լայնորեն կիրառվում են քվանտիտատիվ ֆինանսներում՝
  - Random Forest,
  - Gradient Boosting տիպի մոդելներ (օրինակ XGBoost, LightGBM, CatBoost):
- Այդ մոդելները «սովորում» են պատմական սիգնալներից՝
  - ինչպիսի RSI, վոլատիլություն, կառուցվածք և այլն են ունեցել այն սիգնալները, որոնք հետո հասել են TP, և
  - ինչպիսի հատկանիշներ են ունեցել այն սիգնալները, որոնք արագ հասել են SL:
- Արդյունքում ML մոդելը տալիս է «ում հավանականություն է, որ գինը կշարժվի դեպի վեր» (probability up):

3) Կոմբինացված որոշում
- Վերջնական սիգնալը ձևավորվում է՝
  - կանոնը (moving average) տալիս է հիմնական ուղղությունը (BUY/SELL),
  - ML-ը տալիս է վստահության թվային գնահատական,
  - այս երկուսը համադրվում են և ստացվում է «վստահության» վերջնական score:
- Եթե score-ը բավական բարձր է (օրինակ՝ ≈0.7 և ավելի), ստեղծվում է գործնական սիգնալ (BUY կամ SELL), հակառակ դեպքում՝ NO_TRADE:

Բացի այդ, համակարգը նախազտում է շուկան՝
- պահանջում է, որ վերջին մի քանի րոպեում գնի շարժը բավարար տոկոս լինի (օր. ≥ 2%),
- և որ վերջին մոմի նոտիոնալը (գին * ծավալ) լինի որոշակի նվազագույնից բարձր՝ փափուկ շուկաներից խուսափելու համար:

## 4. Ինչպես են հաշվվում Stop-Loss և Take Profit մակարդակները

Stop-loss և TP-ները որոշվում են ATR-ի հիման վրա՝ որպես «նորմալ» շարժի չափ.

- Սկզբում հաշվում է ATR-ը, բայց օգտագործում է նաև նվազագույն շեմ՝
  - եթե ATR-ը շատ փոքր է, որպես արդյունավետ ATR վերցվում է մոտավորապես գնի 2%-ը,
  - սա արվում է, որ stop-loss-ն ու թարգեթները չլինեն «գրեթե նույն գինը», այլ լինեն իմաստալից հեռավորության վրա:

- Եթե սիգնալը Long է՝
  - Stop-loss = Entry − 1×ATR,
  - TP1 = Entry + 1×ATR,
  - TP2 = Entry + 2×ATR,
  - TP3 = Entry + 3×ATR:

- Եթե սիգնալը Short է՝
  - Stop-loss = Entry + 1×ATR,
  - TP1 = Entry − 1×ATR,
  - TP2 = Entry − 2×ATR,
  - TP3 = Entry − 3×ATR:

Այսինքն՝ մոտավոր Reward:Risk հարաբերությունները լինում են 1:1, 2:1 և 3:1 համապատասխանաբար TP1, TP2 և TP3-ի համար։ Թրեյդերը կարող է ընտրել օգտագործել միայն TP1, կամ սպասել մինչև TP2/TP3՝ ըստ իր ռիսկի ախորժակի։

## 5. Ինչպես է համակարգը ստուգում սիգնալների արդյունքները և «սովորում»

Սիգնալ ուղարկելուց հետո համակարգը մոտավորապես 50 րոպե շարունակ հետևում է հետագա մոմերին.

- Եթե այդ ընթացքում գինն առաջինը դիպչում է Stop-loss-ին → սիգնալը համարվում է պարտված (loss, label=0):
- Եթե գինն առաջինը դիպչում է TP1/TP2/TP3-ից որևէ մեկին → սիգնալը համարվում է հաղթած (win, label=1), և outcome_type դաշտում նշվում է՝ որ նպատակին է առաջինը հասել (TP1, TP2 կամ TP3):
- Բացի այդ, առանձին նշվում է՝ արդյոք երբևէ դիպել է SL, TP1, TP2, TP3 մակարդակներից յուրաքանչյուրին (sl_hit, tp1_hit, tp2_hit, tp3_hit):

Այս ամենը գրանցվում է աղյուսակում (signal_log.csv) և օգտագործվում է՝ ապագայում ML մոդելները վերաուսուցանելու համար։ Այսինքն՝ ծրագիրը պարբերաբար վերլուծում է իր նախկին սիգնալները և փորձում է ավելի լավ տարբերել լավ և վատ setup-երը։

## 6. Ինչ պետք է իմանա թրեյդերը այս սիգնալները օգտագործելիս

- Սիգնալները **խիստ հաշվարկված առաջարկություններ են**, բայց ոչ երաշխիք։ Դրանք հիմնված են պատմական օրինաչափությունների, ոչ թե ապագայի լիարժեք կանխատեսման վրա։
- Համակարգը ավելի հարմար է կարճաժամկետ (intra-day) շարժերի վրա աշխատող թրեյդերի համար, քանի որ արդյունքները ստուգվում են մոտավորապես 50 րոպեի հորիզոնում։
- Թրեյդերը պետք է հաշվի առնի նաև լիկվիդայնությունը, սլիպպիջը, կոմիսիաները և նորությունները, որոնք մոդելը չի «կարդում»՝ ինչպես մարդը կկարդար Twitter-ը կամ նորությունները։
- Stop-loss և TP մակարդակները կառուցված են վոլատիլության (ATR) հիման վրա, ուստի ավելի բարձր վոլատիլության պայմաններում մակարդակները կլինեն ավելի հեռու, իսկ խաղաղ շուկաներում՝ ավելի մոտ։

Այս բացատրությունը պետք է օգնի, որ ոչ ծրագրավորող թրեյդերը հասկանա՝ ինչ լոգիկայով է այս համակարգը ընտրում ուղղությունը, որտեղ է դնում stop-loss-ը և ինչ գաղափարով են ընտրվում նպատակային (TP) մակարդակները։
